Complete Setup Guide:

Create a Windows 11 + Node + NVM + Three.js + Vite + GitHub + AWS Amplify Gen 2 Project

1. Set Up Windows 11 Development Environment

Install Essential Software

  1. Install Visual Studio Code

    • Download from code.visualstudio.com
    • Install recommended extensions:
      • ESLint
      • Prettier
      • JavaScript and TypeScript support
      • Three.js support
      • AWS Toolkit

  2. Install Git for Windows

    • Download from git-scm.com
    • Follow installation instructions
      • Choose "Use Git from Git Bash only" for PATH environment
      • Choose "Checkout as-is, commit as-is" for line endings
      • Enable Git Credential Manager

2. Set Up Node.js with NVM for Windows

  1. Install NVM for Windows

    • Download from GitHub
    • Run the installer and follow the prompts
    • Restart your terminal after installation

  2. Install Node.js LTS version

    • Open PowerShell as Administrator
    • Run: nvm list available to see available versions
    • Install the latest LTS version: nvm install 20.12.2 (or the latest LTS)
    • Set it as default: nvm use 20.12.2
    • Verify installation: node -v and npm -v

  3. Configure npm

    • Set up a global npm directory: npm config set prefix $HOME/.npm-global
    • Add to Windows PATH if needed

3. Initialize Your Project

  1. Create project directory

    powershell
    mkdir my-threejs-project
cd my-threejs-project

  2. Initialize npm project

    powershell
    npm init -y

  3. Set up Vite with Three.js template

    powershell
    npm create vite@latest . -- --template vanilla

  4. Install Three.js and dependencies

    powershell
    npm install three
npm install --save-dev vite-plugin-glsl

  5. Create basic project structure

    /my-threejs-project
├── /src
│   ├── /components
│   ├── /scenes
│   ├── /shaders
│   ├── /textures
│   ├── /models
│   ├── /utils
│   ├── main.js
│   └── style.css
├── /public
│   └── assets
├── index.html
├── vite.config.js
├── package.json
└── README.md

  6. Set up Vite configuration

    Create vite.config.js at the root:
    JavaScript
    import { defineConfig } from 'vite';
import glsl from 'vite-plugin-glsl';
                                    
export default defineConfig({
  plugins: [glsl()],
  base: './',
  build: {
    outDir: 'dist',
    assetsDir: 'assets',
  },
});

  7. Create a basic Three.js scene

    Update src/main.js :
    JavaScript
    import * as THREE from 'three';
import './style.css';

// Create scene
const scene = new THREE.Scene();
scene.background = new THREE.Color(0x1a1a1a);

// Create camera
const camera = new THREE.PerspectiveCamera(
  75,
  window.innerWidth / window.innerHeight,
  0.1,
  1000
);
camera.position.z = 5;

// Create renderer
const renderer = new THREE.WebGLRenderer({ antialias: true });
renderer.setSize(window.innerWidth, window.innerHeight);
document.body.appendChild(renderer.domElement);

// Add a cube
const geometry = new THREE.BoxGeometry(1, 1, 1);
const material = new THREE.MeshNormalMaterial();
const cube = new THREE.Mesh(geometry, material);
scene.add(cube);

// Animation loop
function animate() {
  requestAnimationFrame(animate);
  cube.rotation.x += 0.01;
  cube.rotation.y += 0.01;
  renderer.render(scene, camera);
}

// Handle window resize
window.addEventListener('resize', () => {
  const width = window.innerWidth;
  const height = window.innerHeight;
  renderer.setSize(width, height);
  camera.aspect = width / height;
  camera.updateProjectionMatrix();
});

// Start animation
animate();

  10. Test your setup

    powershell
    npm run dev

  11. Build your project

    powershell
    npm run build

4. Set Up Version Control with GitHub

  1. Initialize Git repository

    powershell
    git init

  2. Create GitHub repository

    1. Go to GitHub and sign in
    2. Click "New repository"
    3. Fill in the details:
      • Repository name: my-threejs-project
      • Description: My Three.js project with Vite
      • Set to public or private
      • Do not initialize with README, .gitignore, or license
      • Click "Create repository"

  3. Link local repository to GitHub

    powershell
    git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/yourusername/your-repo-name.git
git push -u origin main

  4. Password Issues? Use a Personal Access Token (Easy + Fast)

    5. Instead of your GitHub password, you use a token.
    • Create a Personal Access Token
        1. Go to GitHub Tokens Settings
        2. Click Generate new token (Fine-grained or Classic –either is fine)
        3. Give it a name like Amplify Push Token
        4. Scope: check repo permissions (for read/write)
        5. Set an expiration date (1 year, or "No expiration" if you're forgetful 😅)
        6. Click Generate token at the bottom of the page
        7. Copy the generated token and store it in a safe place (you won't be able to see it again)
        8. Use this token as your password when pushing to GitHub from the command line or any Git client.

5. Set Up AWS Amplify Gen 2

  1. Install AWS CLI

    • Download from aws.amazon.com
    • Follow installation instructions
    • Verify installation: aws --version

  2. Configure AWS CLI

    powershell
    aws configure
    • Enter your AWS Access Key ID
    • Enter your AWS Secret Access Key
    • Set default region (e.g., us-east-1)
    • Set default output format (e.g., json)
    You can find your access keys in the AWS Management Console under IAM → Users → Security credentials. To set up user access keys, you may need to create a new user with programmatic access if you don't already have one. View documents

  3. Initialize AWS Amplify Gen 2

    Install AWS Amplify
    powershell
    npm create amplify@latest

  4. Start a local sandbox environment

    powershell
    npx ampx sandbox
    5. This will create a local development environment for your Amplify backend.

6. Set-up AWS Amplify

  1. Login at aws.amazon.com

    • Click on Create New App
    • Select GitHub and then the Next button
    • Select your Repository and hit Next
    • I leave the App Settings page as-is and click Next
    • Usually nothing to change on the Review page either. Click Save and Deploy
    • To view, click on the App link

7. Back to Visual Studio Code

  1. Make edits to the project and Save All

    powershell
    npm run build
    2. I use the Source Control button to push the updates to GutHub. This will automatically trigger a new build in AWS Amplify.
